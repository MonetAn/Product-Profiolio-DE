# Архив SQL-миграций (не применять через CLI)

Файлы здесь **не участвуют** в `supabase db push` / `supabase migration up` — Supabase CLI читает только `supabase/migrations/`.

## Зачем

Схема на prod уже содержит эти изменения (ручные правки, SQL Editor, старые деплои).  
В репозитории оставались «висящие» миграции март–июнь 2026, которые CLI пытался накатить повторно.

**Не переносить обратно в `migrations/` и не запускать на prod без явного решения.**

## Как отличить

- Активные миграции: `supabase/migrations/*.sql`
- Архив: `supabase/migrations_archive/*.sql` (этот каталог)

## Проверка синхронизации

```bash
supabase migration list
```

В колонках Local | Remote не должно быть строк только в Local (кроме новых, ещё не выкатываемых миграций).
