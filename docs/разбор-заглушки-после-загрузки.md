# Разбор: заглушка «Нет инициатив по выбранным фильтрам» после экрана загрузки

## Что происходит

После экрана «Загрузка…» на один кадр (иногда дольше) показывается заглушка **«Нет инициатив по выбранным фильтрам»**, затем появляется нормальный тримап.

## Почему так происходит (причина)

Цепочка такая:

1. **Загрузка**  
   Показывается «Загрузка…», когда:
   - `rawData.length === 0` и
   - `isLoading || (dbData && dbData.length > 0)`.

2. **Приход данных из БД**  
   В `useEffect` по `[dbData, isUsingCSV]` вызывается:
   - `setRawData(result.rawData)` и остальные сеттеры.

3. **Следующий рендер**  
   - `rawData` уже не пустой → условие загрузки ложно.
   - Рендерится основной контент с `BudgetTreemap`.
   - В тримап передаётся:
     - `data={currentRoot}`,
     - `hasData={rawData.length > 0}` → **true**.

4. **Дерево ещё не построено**  
   - `portfolioData` и `currentRoot` обновляются в **другом** `useEffect`:  
     `useEffect(() => { rebuildTree(); }, [rebuildTree]);`
   - Этот эффект выполняется **после** текущего рендера.
   - В этом рендере `currentRoot` всё ещё начальное значение:  
     `{ name: 'Все Unit', children: [], isRoot: true }`.

5. **Логика в TreemapContainer**  
   - `isEmpty = !data.children || data.children.length === 0` → **true** (детей нет).
   - `hasData` из пропсов → **true**.
   - Условие заглушки: `isEmpty && hasData` → **true** → показывается **«Нет инициатив по выбранным фильтрам»**.

6. **Следующий кадр**  
   Выполняется эффект с `rebuildTree()`, обновляются `portfolioData` и `currentRoot`, идёт повторный рендер — тримап уже с данными.

Итого: есть **один (или несколько) кадров**, когда «загрузка» уже скрыта (`rawData` есть), но дерево для тримапа ещё не пересчитано (`currentRoot` без детей). В этот момент тримап считает, что данные есть, а по текущему дереву — пусто, и показывает заглушку по фильтрам.

## Насколько мы в этом уверены

- Цепочка «данные пришли → обновился только `rawData` → рендер с пустым `currentRoot` → эффект с `rebuildTree` позже» однозначно задаётся порядком эффектов и зависимостями в `Index.tsx`.
- Условие заглушки в `TreemapContainer.tsx` (строки 479–495): `isEmpty && hasData` — это ровно тот случай: есть «данные» с точки зрения страницы, но у переданного дерева нет детей.

Вывод: причина — **рассинхрон на один цикл рендера** между появлением `rawData` и обновлением дерева (`currentRoot`/`portfolioData`), а не баг фильтров или пустой ответ API.

## Как помочь с диагностикой (если нужно убедиться у себя)

1. **Логи в консоль**  
   В `Index.tsx` в начале рендера (после хуков) временно добавить:
   ```ts
   console.log('Index render', {
     rawDataLen: rawData.length,
     currentRootChildren: currentRoot.children?.length ?? 0,
     isEmpty: !currentRoot.children || currentRoot.children.length === 0,
     hasData: rawData.length > 0,
   });
   ```
   При проскакивании заглушки должен быть один лог с `rawDataLen > 0` и `currentRootChildren === 0`.

2. **Замедлить сеть**  
   В DevTools → Network → Throttling выставить «Slow 3G», перезагрузить. Окно, где видна заглушка, может стать длиннее — проще поймать и проверить.

3. **React DevTools Profiler**  
   Записать загрузку страницы и посмотреть, сколько рендеров идёт подряд после появления данных: между «Загрузка…» и стабильным тримапом должен быть минимум один лишний рендер с пустым деревом.

После такой проверки можно быть уверенным, что проблема именно в описанном сценарии, и тогда фикс ниже будет уместен.

## Решение

Нужно не показывать контент с тримапом до тех пор, пока дерево не построено после появления `rawData`. Для этого вводится флаг «дерево готово» (`treeReady`):

- Пока `rawData.length > 0` и дерево ещё не пересчитано для этого `rawData` — продолжаем показывать «Загрузка…» (или тот же блок, что и при загрузке).
- Как только сработал эффект с `rebuildTree()` и у нас есть непустой `rawData`, считаем дерево готовым и показываем тримап (или заглушку по фильтрам, если дерево реально пустое из‑за фильтров).

Реализация — в `Index.tsx`: состояние `treeReady` и расширение условия показа блока «Загрузка…» (см. патч в коде).
