# Диагностика анимаций (production)

Чтобы найти причину лагов анимаций на проде:

## 1. Включить диагностику по URL

Откройте приложение с параметром **`?perf=1`** в адресе, например:

- Локально после сборки: `http://localhost:4173/?perf=1`
- На проде: `https://your-domain.com/?perf=1`

При этом:
- В консоли браузера будут логи **длинных задач** (main thread блокировался >50ms) — они и дают фризы.
- В правом верхнем углу появится **FPS-оверлей**; при просадке ниже 30 FPS значение подсвечивается и в консоль пишется предупреждение.

## 2. Запись в Chrome DevTools Performance

1. Откройте приложение с `?perf=1`.
2. Откройте **Chrome DevTools** → вкладка **Performance**.
3. Нажмите **Record** (круглая кнопка).
4. Воспроизведите лаг (zoom в треймапе, переключение фильтров, resize окна).
5. Остановите запись.

На таймлайне ищите:
- **Метки** `treemap-animation-drilldown`, `treemap-drilldown-start`, `treemap-drilldown-end`, `treemap-animation-filter`, `treemap-animation-resize` — по ним видно момент анимации.
- **Длинные жёлтые блоки** (Long Task) во время анимации — это и есть причина фриза.

## 3. После диагностики

Параметр `?perf=1` можно не передавать — тогда никакой оверлей и лишние логи не показываются. Метки `performance.mark` в коде остаются (они почти не нагружают приложение).
