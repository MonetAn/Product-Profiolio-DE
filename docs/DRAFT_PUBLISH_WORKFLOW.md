# Черновик → пересчёт → публикация (идея для следующих итераций)

## Что ты описал

1. Лидер заходит, вводит коэффициенты.
2. Нажимает **«Рассчитать»** (или аналог): уходит в Google Sheets / формулы, возвращаются **промежуточные** стоимости.
3. Смотрит цифры, меняет коэффициенты, снова пересчитывает — столько раз, сколько нужно.
4. Пока это **черновик**: другие пользователи **не видят** эти суммы в «боевом» портфеле.
5. **«Сохранить / опубликовать»** — только тогда данные становятся **официальными** и видны всем.

Это сделать **можно**. Ниже — модель данных и вопросы, которые нужно закрыть перед UI.

---

## Предлагаемая модель (концепт)

| Слой | Где хранить | Кто видит |
|------|-------------|-----------|
| **Черновик** | Например `quarterly_data.sheet_preview` или отдельная таблица `initiative_sheet_drafts` (user_id, payload, updated_at) | Только автор черновика (и супер-админ) |
| **Опубликовано** | Текущие `quarterly_data.cost`, `sheet_out_itog_2025` | Все с доступом к портфелю |

Поток:

- **Пересчёт** → импорт с OUT пишет только в **preview** (или в JSON с флагом `published: false`).
- **Опубликовать** → копирование preview → в основные поля (`cost`, `sheet_out_itog_2025`) + запись в `activity_events` / аудит.

Альтернатива проще на первом шаге: два набора ключей в одном JSON — `sheet_out_itog_2025_draft` и при публикации merge в `sheet_out_itog_2025` + `cost`.

## Вопросы, которые нужно решить

1. **Область черновика** — на одного лидера, на команду, на весь юнит? Что если два лида правят одну инициативу?
2. **Один глобальный черновик или у каждого свой?** (обычно: свой черновик + конфликт при публикации если кто-то уже опубликовал новее.)
3. **Google Sheets** — один файл на всех или персональный «песочница»? Сейчас один spreadsheet: для изоляции черновика либо отдельные вкладки/файл, либо расчёт только в приложении до публикации.
4. **Кто может «Опубликовать»** — только админы, или лидер своей команды?
5. **Нужна ли история** — версии, откат к прошлой публикации?

## Связь с текущим кодом

Сейчас **«Стоимость ← OUT»** сразу пишет в **опубликованные** поля. Следующий шаг по продукту:

1. Добавить миграции/поля под **draft** (по ответам на вопросы выше).
2. Разделить Edge или флаги тела запроса: `mode=draft` vs `mode=publish`.
3. UI: кнопки «Предпросмотр / Рассчитать» и «Опубликовать», бейдж «есть несохранённый черновик».

Когда зафиксируете ответы на блок «Вопросы» — можно проектировать конкретные таблицы и экраны вместе с UI.
