# Анимация следов (карточка «Заполни по шагам»)

## Что читается как «следы», а что нет

- **Хорошо:** силуэт лапы **сразу финального размера**, появление через **opacity** (fade-in) + небольшой **сдвиг по Y** (как лёгкое «приземление»), следы идут **по очереди** (stagger).
- **Плохо для отпечатка:** сильный **scale от почти нуля** — это похоже на UI-микроинтеракцию, а не на отпечаток на поверхности.

## Практики из индустрии

1. **Stagger / overlapping action** — задержка между элементами по порядку тропы (как «шаги»). Общая идея хореографии motion в UI (в т.ч. рекомендации к последовательной анимации списков).
2. **Opacity как основной канал** для «появился след» — быстро читается и не ломает форму.
3. **Сложные тропы по траектории** часто делают в **GSAP / Motion** с путём и поворотом каждого шага; для статичной дуги из 5 иконок достаточно **CSS** + `animation-delay` по `nth-child`.
4. **Тач:** при отсутствии `:hover` следы показываются статично (`@media (hover: none)`).

## Текущая реализация (кратко)

- Силуэт: SVG Repo **paw-print-bold** (path в `AdminScenarioIllustrations.tsx`, копия `public/admin-icons/paw-print-bold.svg`).
- Ключевые кадры: `admin-scenario-paw-fade-in` в `src/index.css`.
- Длительность одной лапы **~0,52s**, шаг между стартами **0,16s**, easing **`ease-out`**.
- Без hover: `opacity: 0` на `.admin-scenario-paw-wrap`; снятие hover сбрасывает к скрытому состоянию (повтор при следующем наведении).
