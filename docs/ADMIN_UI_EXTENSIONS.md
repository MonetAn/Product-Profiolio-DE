# Расширения UI админки (Dodo Engineering)

Краткий гайд для новых экранов и для AI-агента. Полный план: `ADMIN_DODO_ENGINEERING_REDESIGN.md`. **Цвета и тема админки:** `ADMIN_UI_COLORS_AND_THEME.md`.

## 1. Тема

- Все маршруты `/admin*` оборачиваются в **`AdminRoute`** → на `<html>` вешается **`admin-engineering`**.
- Цвета интерфейса — **CSS-переменные shadcn** (`bg-background`, `text-primary`, `border-border`, …). В админке **`--primary` = индиго DE**, не оранжевый дашборда.
- **Primary (индиго):** главные кнопки, активные табы, **настоящие ссылки** (внешний Doc и т.п.). Не красить весь текст таблицы в primary.
- **Таблица «Заполнение»:** подсветки неполных полей/строк убраны (временно); позже отдельный UX для валидации.
- **Красный:** опасные действия (удаление), не оранжевый.
- **Внимание / несоответствия / подсветка полей:** **голубой** — классы Tailwind `blue-*` (или `hsl(var(--admin-attention))` в CSS под `html.admin-engineering`). **Не** подменять ими семантические шкалы (исключение: **`AdminQuickFlowTeamQuarterPicker`** — зелёный / янтарь / красный / серый, см. `ADMIN_UI_COLORS_AND_THEME.md`).

## 2. Типографика

- **Основной текст и таблицы:** Inter (наследуется с `body`).
- **Крупные заголовки экранов:** `font-juneau font-medium` (+ при необходимости `tracking-tight`).

## 3. Вне скоупа

- Графики и публичный дашборд — отдельная палитра (оранжевый Brands ок там).

## 4. Иллюстрации сценария

- Стили в `index.css`: классы `admin-scenario-*`; акценты через **`hsl(var(--primary))`**.

## 5. Чеклист перед мержем нового админ-экрана

- [ ] Нет оранжевого Brands в primary/CTA админки; валидация общих экранов — `blue-*` или `--admin-attention` (янтарь только там, где задана отдельная шкала, напр. выбор кварталов усилий).
- [ ] Кнопки CTA и активные табы на токенах `primary`.
- [ ] Заголовок страницы в Juneau по правилам выше.
