# План: убрать ошибки Google Sheets + почему в чате Cursor «всё долго крутится»

## 1. Про «бесконечную» работу в чате (операция в Cursor)

Это **не баг приложения**. В чате агент часто:

- запускает **долгие команды** (`supabase functions deploy` может идти **много минут**, особенно при медленном интернете);
- делает **много шагов подряд** (чтение файлов, деплой, проверки);
- при **TLS timeout** к `api.supabase.com` команда падает и её **повторяют** — снова долго.

**Как снизить боль:**

- Делить задачи: сначала «только проверь `.env`», отдельно «только деплой одной функции».
- Тяжёлые команды **выполнять самому в терминале** и в чат кидать **только вывод** (последние 30 строк).
- Если команда висит **>10–15 минут** — **Ctrl+C** и повторить в другой сети (без VPN / с телефона как точка доступа).

## 2. План по шагам: чтобы ошибки в админке ушли

Идея: каждый шаг убирает один класс ошибок. Идти **сверху вниз**.

### Шаг A — Один и тот же проект Supabase везде

| Где смотреть | Что должно совпадать |
|--------------|----------------------|
| Файл `.env` в корне проекта | `VITE_SUPABASE_URL` = `https://<ТВОЙ_REF>.supabase.co` |
| Dashboard → Edge Functions | Тот же `<ТВОЙ_REF>` в URL |
| Список функций | Есть **обе**: `sheets-export-initiatives` и `sheets-import-from-sheet` |

Если в `.env` **другой** ref, чем в Dashboard — кнопки бьют «не туда» (404 / странные ошибки).

**Действие:** открыть `.env`, сравнить URL с адресной строкой проекта в Supabase.

---

### Шаг B — Обе функции задеплоены

Раньше была только выгрузка — импорт без деплоя даст ошибку на второй кнопке.

**Действие:** Dashboard → Edge Functions → убедиться, что в списке **две** функции. Если одной нет — в терминале из папки проекта:

```bash
supabase functions deploy sheets-import-from-sheet
supabase functions deploy sheets-export-initiatives
```

(после `supabase link` и `supabase login`.)

---

### Шаг C — Secrets в Supabase

Dashboard → Edge Functions → **Secrets**:

- `GOOGLE_SERVICE_ACCOUNT_KEY` — полный JSON ключа.
- `GOOGLE_SHEETS_SPREADSHEET_ID` — ID таблицы из URL.

Без них функция ответит 500 с текстом про missing secrets.

---

### Шаг D — Доступ Google Таблицы к сервисному аккаунту

В таблице: **Доступ** → email из поля `client_email` в JSON (…`@…iam.gserviceaccount.com`) → **Редактор**.

Иначе Google API вернёт ошибку доступа при выгрузке/чтении.

---

### Шаг E — Ты админ в приложении

Вызов идёт с твоим JWT; внутри функции проверяется `get_my_access().is_admin`.

Если не админ — будет 403 / сообщение про доступ.

---

### Шаг F — Ошибка «Failed to send a request to the Edge Function»

Это **сетевой сбой до Supabase** (не логика таблицы).

**Действия:**

1. Обновить страницу, выйти/войти снова.
2. Повторить кнопку (в коде уже есть повторы + fallback на прямой `fetch` — подтяни последний код из репозитория).
3. Другая сеть / отключить VPN / другой браузер без расширений.
4. DevTools → **Network** → клик по запросу `.../functions/v1/sheets-export-initiatives` — статус, красный ли запрос, CORS.

**CORS: `x-supabase-client-platform is not allowed`**

Новые версии `@supabase/supabase-js` добавляют заголовок `x-supabase-client-platform`. В ответе **OPTIONS** Edge Function в `Access-Control-Allow-Headers` его нужно разрешить (или отразить `Access-Control-Request-Headers` из запроса). В репозитории это сделано в `supabase/functions/_shared/cors.ts` — после правки **обязательно заново задеплой обе функции** (Шаг B).

---

### Шаг G — Ошибка после того как запрос дошёл (HTTP 4xx/5xx)

Смотреть **тело ответа** в Network или **Logs** в Supabase → Edge Functions → выбрать функцию → Logs.

Типично: неверный JSON ключа, нет прав на лист, ошибка Google OAuth.

---

## 3. Минимальный чеклист (галочки)

- [ ] `.env`: URL = тот же проект, что в Dashboard  
- [ ] В Edge Functions видны **обе** функции  
- [ ] Secrets: ключ + spreadsheet id  
- [ ] Таблица расшарена на SA  
- [ ] Ты залогинен и ты **admin**  
- [ ] Сеть/VPN не режет `*.supabase.co`  
- [ ] Нет CORS на `x-supabase-client-platform` → последний `cors.ts` + редеплой функций  

После этого кнопки **«Выгрузить в лист»** / **«Импорт с листа»** должны либо работать, либо показывать **осмысленный** текст ошибки (не только «Failed to send…»).

---

## 4. Что не нужно ожидать от чата

- Мгновенного деплоя: CLI и сеть реально могут занимать минуты.  
- Деплоя без твоего интернета: агент на твоей машине упирается в **твой** TLS/VPN.  

Если что-то из чеклиста не сходится — исправь это **вручную** по шагам A–G, затем снова проверь кнопки.
